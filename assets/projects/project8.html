SSAFY 12기 공통 프로젝트로 개발한 블록체인 기반 반려동물 의료정보 관리 시스템입니다. 프라이빗 이더리움 네트워크(Besu)를 활용해 가스비 없이 트랜잭션을 처리하고, 보호자가 진료 기록을 블록체인에 저장·조회하며 병원별 접근 권한을 직접 제어하는 구조를 구현했습니다. 블록체인 및 프론트엔드를 담당했습니다.

서비스가 필요한 이유

  • 병원을 변경할 때마다 반려동물의 과거 진료 내용을 직접 설명해야 하는 불편함이 존재합니다.
  • 보호자가 예방접종 내역이나 병력 정보를 잊어버리면 정확한 진료가 어려워질 수 있습니다.
  • KKUKKKUK은 기록을 분실 걱정 없이 블록체인에 보관하고, 필요 시에만 병원과 안전하게 공유할 수 있습니다.

핵심 설계 — DIDRegistry 컨트랙트

기존 의료 시스템에서 진료 기록의 소유권은 병원에 있었습니다. 이 프로젝트는 DID(탈중앙화 식별자) 개념을 반려동물에 적용해, 각 반려동물이 고유한 이더리움 주소를 가지고 보호자가 병원별 접근 권한을 직접 제어하는 DIDRegistry 컨트랙트를 설계했습니다.

데이터 구조

// DIDRegistry.sol — 핵심 상태 변수

// 반려동물 주소 => 소유자(보호자) 주소
mapping(address => address) public owners;

// 반려동물 주소 => 접근 권한이 있는 병원 주소 배열
mapping(address => address[]) public hospitals;

// 반려동물 주소 => 속성명 => 속성 구조체 (의료 기록, 프로필 등 모든 데이터를 key-value로 저장)
mapping(address => mapping(string => Attribute)) private attributes;

// 반려동물 주소 => 병원 주소 => 공유 계약 (시간 제한 접근)
mapping(address => mapping(address => SharingAgreement)) public sharingAgreements;

struct SharingAgreement {
    bool   exists;
    string scope;          // 공유 범위 ("all", "basic", "medical")
    uint   createdAt;
    uint   expireDate;     // 최소 1일 ~ 최대 30일
    bool   notificationSent;
}

// 공유 기간 상수
uint public constant MIN_SHARING_PERIOD = 1 days;
uint public constant MAX_SHARING_PERIOD = 30 days;
uint public constant NOTIFICATION_THRESHOLD = 3 days; // 만료 3일 전 알림

반려동물 등록 — 단일 트랜잭션으로 속성 일괄 저장

function registerPetWithAttributes(
    address petAddress,
    string memory name,
    string memory gender,
    string memory breedName,
    string memory birth,
    bool flagNeutering,
    string memory speciesName,
    string memory profileUrl
) public {
    require(owners[petAddress] == address(0), "Pet address is already registered");
    owners[petAddress] = msg.sender;
    _addPetToOwner(msg.sender, petAddress);

    uint validity = 100 * 365 * 24 * 60 * 60; // 100년 유효

    _setAttributeInternal(petAddress, "name", name, validity);
    _setAttributeInternal(petAddress, "gender", gender, validity);
    _setAttributeInternal(petAddress, "breedName", breedName, validity);
    _setAttributeInternal(petAddress, "flagNeutering", flagNeutering ? "true" : "false", validity);
    // ...

    emit PetRegistered(petAddress, msg.sender, block.timestamp);
}

시간 제한 공유 계약 — QR 스캔 시 실행되는 트랜잭션

function addHospitalWithSharing(
    address petAddress,
    address hospitalAddress,
    string memory scope,
    uint sharingPeriod   // 1일 ~ 30일
) public {
    require(msg.sender == owners[petAddress], "Only owner can add hospital access");
    require(sharingPeriod >= MIN_SHARING_PERIOD && sharingPeriod <= MAX_SHARING_PERIOD);

    uint expireDate = block.timestamp + sharingPeriod;

    sharingAgreements[petAddress][hospitalAddress] = SharingAgreement(
        true, scope, block.timestamp, expireDate, false
    );

    emit SharingAgreementCreated(petAddress, hospitalAddress, scope, expireDate, block.timestamp);
}

// 만료 확인 — view 함수이므로 가스 없이 로컬 호출
function hasAccess(address petAddress, address user) public view returns (bool) {
    if (owners[petAddress] == user) return true;

    SharingAgreement memory agreement = sharingAgreements[petAddress][user];
    if (agreement.exists) {
        return block.timestamp <= agreement.expireDate; // 만료 시 false 반환
    }
    // ...
}

의료 기록 저장 — Hub-Spoke 구조

진료 기록은 JSON 문자열로 직렬화되어 setAttribute()를 통해 온체인 저장됩니다. 최초 기록(medical_record_*)이 허브가 되고, 이후 수정 기록(medical_update_*)이 원본 키를 참조하는 Spoke 방식으로 연결됩니다. 블록체인의 불변성 때문에 삭제는 isDeleted: true 필드를 추가하는 소프트 삭제로 처리했습니다. 

// 의료 기록 키 구조
// 원본: "medical_record_{timestamp}_{hospitalAddress}"
// 수정: "medical_update_{timestamp}_{hospitalAddress}"

// Hub-Spoke: 원본 기록 키 => 연결된 수정 기록 키 배열
mapping(string => string[]) private medicalRecordUpdates;

// 소프트 삭제 — 블록체인 불변성으로 실제 삭제 불가, isDeleted 플래그로 처리
function softDeleteMedicalRecord(address petAddress, string memory recordKey) public {
    // ...
    string memory updatedData = replaceJsonBoolValue(recordData, "isDeleted", "true");
    updatedData = addJsonFieldValue(updatedData, "deletedAt", uint2str(block.timestamp));
    updatedData = addJsonFieldValue(updatedData, "deletedBy", addressToString(msg.sender));
    setAttribute(petAddress, recordKey, updatedData, expireDate - block.timestamp);
}

스마트컨트랙트 설계 및 배포

Solidity로 반려동물 의료 기록 저장·조회·권한 관리 로직을 구현하고 Remix IDE를 통해 프라이빗 이더리움 네트워크에 배포했습니다. 반복 테스트를 위해 배포 스크립트를 Hardhat으로 자동화하고, 컨트랙트 ABI를 프론트엔드에서 직접 임포트하는 방식으로 연동 구조를 단순화했습니다.

MetaMask 연동 DApp 구현

React에서 ethers.js를 사용해 MetaMask 지갑 연결, 트랜잭션 서명, 컨트랙트 함수 호출을 구현했습니다. 지갑 미연결 상태, 네트워크 불일치, 트랜잭션 거부 시 사용자 친화적인 피드백을 제공하도록 예외 처리를 작성했습니다.

트랜잭션 성능 최적화

진료 기록 조회가 잦은 병원 측 케이스에서 매번 call()을 트랜잭션으로 처리하는 것이 불필요한 가스 소모로 이어진다는 점을 발견했습니다. 읽기 전용 함수에 view 키워드를 적용하고 로컬에서 직접 호출하도록 변경하여 조회 호출이 체인에 기록되지 않고 즉시 반환되도록 개선했습니다.
